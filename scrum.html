Scrum

Marco ágil de trabajo. No es proceso ni metodología, los procesos y metodologías definen los mecanismos o procesos a seguir para el logro de un objetivo.
Por otra parte se puede entender Scrum como contenedor para otras prácticas o métodos ágiles.

Es un proceso de gestión que reduce la complejidad de productos para satisfacer las necesidades de los clientes. => "conjunto de buenas prácticas para trabajar colaborativamente, en equipo y obtener el mejor resultado posible de un proyecto"

Forma de trabajo

Se realizan entregas parciales y regulares del producto final, priorizadas por el beneficio que aportan al cliente. Se utiliza donde se necesitan resultados pronto, los requisitos son cambiantes o poco definidos, donde la innovación, flexibilidad y productividad son fundamentales.

¿Cuándo se utiliza?

  • cuando no se está entregando al cliente lo que busca
  • cuando las entregas tardan demasiado y los costes se disparan
  • cuando es necesario identificar y solucionar ineficiencias sistemáticamente
  • cuando se busca obtener resultados rápido

Artefactos

son 3 que se refieren a elemetnos físicos al aplicar Scrum =>

  1. PRODUCT BACKLOG :

    requerimientos, casos de uso, dependencias. Fuente principal de información sobre el producto;

  2. SPRINT BACKLOG :

    se visualiza el trabajo y lo gestiona el equipo de desarrollo, quien lo actualiza durante el curso del proyecto, a través de los dailys. Permite analizar hasta donde se cumplió el objetivo de cada sprint y que se podría eliminar

  3. INCREMENTO :

    suma de todas las tareas, casos de uso y cualquier elemento desarrollado y que será puesto a disposición del usuario final. Así se cosntruye software de manera iterativa e incremental
